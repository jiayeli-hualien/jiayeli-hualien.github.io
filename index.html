鏈結串列基本知識

鏈結串列，英文 linked list，是一種資料結構， list 中有多個節點，每個節點當中至少會有兩個成員變數:

  • data/key: 儲存資料的空間，一般會用 int 當作範例，實務上可能會遇到自訂的 struct / pointer / smart pointer ， 或是更瘋狂的 Linux kernel 當中需要利用 macro container_of() 來算回 data 的 address
  • next: 指出下一個節點的 address

下圖是一個鏈結串列的範例，有三個 node 串在一起，上面存的 data 依序是 1 3 5。
若想要存取資料，需要從 head 進入，依序走過所有的 node ，直到抵達 null(空指標)。

相關操作與時間複雜度

操作 時間複雜度 說明
search O(n) 若想找到對應的資料或是 key，需要走過整個 list
insert O(1) (已先做好 search) 不需要再做 search，馬上 insert 的狀況。
只需要操作一個node，或是偶爾需要再操作heed/tail。執行時間有固定的上限。
實務上會需要做 searcrh + insert 所以可能會需要 O(n)
delete O(1) (不含搜尋) 同上。實務上會需要先搜尋再刪除。
append O(1) append 到 head 或是 tail，需要的時間是 O(1)。

n 表示 list 長度

我這裡因為面試被 linked list 問細節問倒了，所以想手動刻一套實作並丟到 UVa 10935 題目 AC 驗證實作正確性。

該題目需要做以下兩個動作

  • 有 N 張牌
  • 從牌庫頂部抽一張牌並丟掉，並印出丟掉哪一張牌(編號 1~N)
  • 再將現在牌庫頂部的牌移動到牌庫底部
  • 持續以上動作直到牌庫剩一張牌，印出該牌的編號
    以上操作只涉及牌頂以及牌底的操作，且牌庫內的牌會維持順序，非常適合用 list 或是 queue 實作:
